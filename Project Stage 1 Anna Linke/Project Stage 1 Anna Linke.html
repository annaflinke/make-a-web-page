Lesson 1: The Basics of the Web and HTML

HTML - Hyper Text Markup Language

HTML documents are the foundation of the Word Wide Web.

HTML contains of:
- Text (what you see)
- Markup (what it looks like)
- References to other documents (e.g. pictures and videos)
- Hyperlinks to other webpages

Markup (Tags and Attibutes)

Markup renders plain text. Specific tag elements can be used to display different text styles and formatting.

Tags

Most tags have an opening tag and a closing tag. Two common tags are the <b> and the <em> tag:

The <b> tag is used to change the fond to bold and is closed with </b>

The <em> tag is used to change the fond to italic and is closed with </em>

Attributes

A tag can have multiple attributes. An example of a tag using an attribute is the anchor tag. The anchor tag is used to create links to other websites.
This link explains how the achor tag works.

Adding Images

Images can be added by using the following tag:
<img src="URL" alt="text">

SRC stands for source and contains the link to where the picture is stored.
ALT the text in the quotes is displayed if the piture cannot be found or read out if reading is enabled.

Inline and Block Tags

Inline tags are text elements. For example the <br> tag can be used in between text to start a new line.
Block tags create a box around the element and can have hight and width. For example the <p> tag creates a new paragraph box that needs to be closed with a closing tag (</p>).

Lesson 2: Creating a Structured Document with HTML

Main Elements to Create Websites

HTML defines the structure of a website.
CSS controls the style of a website.
Javascript adds functionality to a website.

Developer Tools

Browser developer tools can be used to analyse websites. For example this includes the following options:

Structure: the panel in the developer tools shows how the browser interprets the structure of a webpage.
Tags: the structure can be expanded until the HTML code for each element is visible.
Style section: when an element is selected the style section shows the style of the element (color, fond size, font family, etc.).

Tree-like Structure (Document Object Model)

The tree-like structure (also known as tree diagramm) is a way to display the hierarchical nature of a structure/document in graphical form.

In programming this is also called DOM (Document Object Model). HTML elements often contain more elements; the browser displays this as boxes inside boxes.

Boxifying Design

Every element on a website is rectangular.

A website usually has three main boxes each containing more boxes:

- Header box
- Middle box
- Footer box

The first step to develop a website is to devide the page into boxes. This is usually done by using <div> or <span> tags.

Lesson 3: Adding CSS Style to HTML Structure

CSS - Cascading Style Sheets

CSS allows programmers to avoid repetition.

Multiple cascading style sheets can be used to define the style of an HTML document.

Inheritance

Inheritance is a key feature of CSS and is based on an ancestor-decendent relationship. Decendent elements automatically inherit text-related properties (like color, font and letter-spacing), however box related properties (like background, border or position) are not inherited.

Selectors and Styles

Selectors can be tags or classes.

The selector defines to which elements a particular style will be applied.

Style (rules) examples:
- font-style: italic, normal, etc.
- font-weight: bold, normal, etc.
- color: black, white

The style rules are defined in curley backets using the following syntax {attribute: value}.

Ways to Link CSS into a Document

Method 1

Write CSS in the <head> of your HTML document.

This method is usfull for very small projects.
Simply add a <style> element containing the CSS in the <head> element of the HTML document.

Method 2

Link your HTML document to a seperate CSS file.

This method helps to stay organized when working on bigger projects.
Add a <link> element containing the link to the CSS file in the <head> element of the HTML document.

Method 3

Write style inline with HTML.

This method is not recommended as it leads to repetitive code and makes websites harder to maintain.
The style is directly added as an attribute in the <div> element.
